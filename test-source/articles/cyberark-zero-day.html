Hackers exploit CyberArk PAM zero-day to steal privileged credentials

Threat actors are actively exploiting a critical zero-day vulnerability in CyberArk Privileged Access Manager, tracked as CVE-2026-21001, to extract privileged account credentials from enterprise vaults, security researchers warned on Monday.

The flaw allows an unauthenticated remote attacker to bypass vault authentication and retrieve stored credentials, including domain administrator accounts. Successful exploitation gives attackers the keys to an organization's most sensitive systems.

Researchers at Halcyon Labs said they observed exploitation attempts against internet-exposed PAM instances beginning over the weekend, with at least a dozen organizations confirmed compromised so far. Attackers used the stolen credentials for lateral movement within minutes of initial access.

CyberArk has released an emergency patch and urges all customers to update immediately. Organizations that cannot patch should restrict network access to the vault web interface and rotate all stored credentials as a precaution.

CISA is expected to add the vulnerability to its Known Exploited Vulnerabilities catalog this week, which would require federal agencies to patch within an accelerated timeframe.
